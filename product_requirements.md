# AI术语关联图谱产品需求文档

## 1. 产品概述

### 1.1 产品背景
当前AI领域术语层出不穷，包括过去的经典术语和当下最新流行的术语，对于小白用户来说理解这些术语及其关联关系非常困难。需要一个可视化的工具来展示AI术语之间的关联关系，并提供通俗易懂的解释。

### 1.2 产品目标
创建一个AI术语关联图谱网页，帮助用户理解复杂的AI术语及其相互关系，降低AI学习门槛。

### 1.3 目标用户
- 个人使用，用于学习和理解AI术语
- 需要本地运行，暂不考虑多人使用或在线部署

## 2. 核心功能

### 2.1 术语图谱展示
- 通过图谱的形式将AI术语节点关联起来
- 可视化展示术语之间的层级关系和关联关系
- 支持图谱的缩放、拖拽、平移等交互操作

### 2.2 术语解释交互
- 通过悬停交互展开AI术语的通俗介绍
- 解释内容通俗易懂，避免专业术语堆砌
- 支持点击查看更详细的信息

## 3. AI术语分类结构

### 3.1 主节点1：LLM (大语言模型)
**通俗解释**：AI大脑，它是一个阅读了互联网上几乎所有文本的"超级博学家"。它不仅能聊天，还能写代码、推理、翻译等等。

#### 次节点1：多模态
**通俗解释**：大模型不仅懂文字，还懂视频、音频、图表。就像人不仅能看书，还能看图、听音乐、看电影一样，AI也能同时处理多种类型的信息。

#### 次节点2：提示词
**通俗解释**：怎么描述才能让AI听得懂，干得好。就像跟人说话一样，怎么说、怎么问，AI的回答质量就会不一样。好的提示词能让AI更好地理解你的需求。

- 次次节点1：提示词工程（专门研究怎么写好提示词的技术）
- 次次节点2：结构化提示词（用固定格式写提示词，让AI更容易理解）

#### 次节点3：上下文
**通俗解释**：就像人的"工作记忆"。上下文越长，你能跟它聊的天就越长，它越不会聊着聊着就忘了前面说过什么。就像你跟人聊天，对方记得你们之前聊过什么。

#### 次节点4：幻觉
**通俗解释**：当AI不知道答案时，它可能会编造一个看起来非常真实但实际上错误的事实。AI有时候会一本正经胡说八道。就像考试时不会做，但为了得分硬编了一个看起来很合理的答案。

### 3.2 主节点2：模型产品
**通俗解释**：这些是你能直接使用的AI产品，就像不同品牌的手机，各有各的特点。

#### 次节点1：国外模型
**通俗解释**：国外公司开发的AI产品

- 次次节点1：GPT系列（OpenAI开发的，最早火起来的AI）
- 次次节点2：Claude系列（Anthropic开发的，以安全性著称）
- 次次节点3：Gemini（Google开发的，搜索功能很强）
- 次次节点4：Llama（Meta开源的，免费给开发者用）

#### 次节点2：国内模型
**通俗解释**：国内公司开发的AI产品

- 次次节点1：文心一言（百度开发的，中文理解能力强）
- 次次节点2：通义千问（阿里巴巴开发的，办公场景好用）
- 次次节点3：Kimi（月之暗面开发的，长文本处理能力强）
- 次次节点4：DeepSeek（深度求索开发的，开源且性价比高）
- 次次节点5：智谱清言（智谱AI开发的，学术研究能力强）
- 次次节点6：豆包（字节跳动开发的，年轻化、娱乐化）

### 3.3 主节点3：数据增强与优化
**通俗解释**：就像给AI补充知识，让它更专业、更懂你的需求。

#### 次节点1：RAG (检索增强生成)
**通俗解释**：AI开卷考。在回答问题前，先去指定的知识库（如公司文档、最新新闻）里查找资料，然后基于查到的资料回答，而不是仅靠脑子背。用来解决AI不知道公司内部数据或特定领域发生事情的问题。

- 次次节点1：向量数据库（AI专用的图书索引系统。它把文字、图片转换成数学上的"向量"（一串数字），让AI能通过"意思相似"而不是"关键词匹配"来快速找到资料。传统搜索是找"包含'苹果'这个词的文件"；向量搜索是找"关于水果、科技公司或牛顿故事"的文件，哪怕文件里没出现"苹果"二字。）
- 次次节点2：知识库（存放AI可以查阅的资料的地方，就像图书馆）

#### 次节点2：微调
**通俗解释**：给AI大脑做专科培训。在通用大模型的基础上，用特定领域的数据（如医疗、法律）再训练一次，让它成为该领域的专家。LLM是"全科医生"，微调后变成了"心脏外科专家"。

- 次次节点1：全量微调（从头到尾重新训练一遍，效果好但成本高）
- 次次节点2：LoRA微调（只训练一小部分参数，成本低但效果也不错）
- 次次节点3：PEFT（参数高效微调）（用最少的训练量达到最好的效果）

#### 次节点3：模型压缩
**通俗解释**：让大模型变小，运行更快。就像把高清视频压缩成标清，画质损失不大但文件小了很多。

- 次次节点1：量化（把数字精度降低，模型变小但影响不大）
- 次次节点2：蒸馏（把大模型的知识教给小模型）
- 次次节点3：剪枝（把模型中不重要的部分删掉）

### 3.4 主节点4：智能体
**通俗解释**：能自主干活的AI。以前的AI是"顾问"（只给建议），Agent是"助理"（给建议并帮你把事办了）。

#### 次节点1：核心能力
**通俗解释**：智能体必备的本领

- 次次节点1：函数调用（AI大脑决定"我要查天气"，通过函数调用这个"手"去点击了天气软件的查询按钮。）
- 次次节点2：工作流（AI任务流水线，将一个大任务拆解成多个步骤，按顺序或条件自动执行。就像工厂的组装线。第一步AI写大纲 -> 第二步AI查资料 -> 第三步AI写正文 -> 第四步AI配图。）
- 次次节点3：规划能力（AI能自己想清楚先做什么后做什么）
- 次次节点4：记忆能力（AI能记住之前的对话和任务，不会忘事）

#### 次节点2：技能生态
**通俗解释**：就像给智能体准备各种工具，让它能干更多事。

- 次次节点1：Skills（技能包）（如果Agent是一个全能员工，那么Skill就是他掌握的具体单项手艺，比如"查天气"、"读PDF"、"搜索谷歌"、"发送钉钉消息"。现在的开发模式不再是训练一个大模型，而是像搭积木一样，把各种现成的Skills组装给Agent。开发者社区充满了共享的Skill库（类似App Store，但是是给AI用的））
- 次次节点2：MCP（模型上下文协议）（一种统一的沟通标准协议，让任何AI模型都能轻松连接到任何数据源或工具。就像USB接口，不管什么设备都能插。）
- 次次节点3：插件系统（给AI安装各种插件，扩展它的功能）

#### 次节点3：开发工具
**通俗解释**：用AI来写代码的工具

- 次次节点1：Claude Code（编程最强智能体，能帮你写代码、改代码、运行代码）
- 次次节点2：Cursor（vibe coding编程开发环境，用AI辅助写代码，体验很酷）
- 次次节点3：Trae（新一代AI原生的集成开发环境，国产IDE，算是Cursor的竞品）
- 次次节点4：OpenClaw（今年开年火的一塌糊涂的AI编程工具）

## 4. 交互设计

### 4.1 图谱交互
- 鼠标悬停：显示术语的通俗解释（tooltip形式）
- **鼠标点击节点（第一次）**：筛选显示该节点及其直接相连的节点
- **主节点第二次点击**：展开显示该主节点下的所有关联节点（包括间接关联的所有子节点）
- **次节点/次次节点第二次点击**：恢复显示所有节点
- **第三次点击主节点**：恢复显示所有节点
- 点击空白处：恢复显示所有节点
- 拖拽节点：调整节点位置
- 滚轮缩放：放大或缩小图谱
- 双击空白处：重置图谱视图

### 4.2 节点样式
- 主节点：较大尺寸，醒目颜色
- 次节点：中等尺寸，区分颜色
- 次次节点：较小尺寸，统一颜色
- 关联线：使用箭头表示层级关系

### 4.3 信息展示
- 悬停提示框（Tooltip）：显示术语名称和通俗解释，支持自动换行，限制最大宽度
- 节点筛选：点击节点后，只显示该节点及其直接相连的节点，方便查看关联关系
- 全图恢复：点击空白处或再次点击已选中节点，恢复显示所有节点

## 5. 技术要求

### 5.1 前端技术
- **前端框架**：Vue 3（通过CDN引入）
- **图谱可视化库**：ECharts 5.x（通过CDN引入）
- **开发语言**：纯JavaScript（ES6语法）
- **样式**：CSS3
- **运行方式**：本地运行，支持浏览器直接打开HTML文件

**技术栈选择理由**：
- Vue 3学习曲线平缓，适合个人快速开发
- ECharts专门支持关系图，内置Graph图表类型，交互丰富
- 中文文档完善，学习成本低
- 性能优秀，支持100+节点流畅渲染
- 可以通过CDN引入，无需构建工具，双击HTML文件即可运行

### 5.2 数据存储
- **数据格式**：JavaScript对象（内嵌在app.js中）
- **存储方式**：直接嵌入到JavaScript文件中
- **数据结构**：包含nodes（节点）和links（关联）两个数组
- **更新方式**：直接编辑app.js文件中的graphData对象

**数据存储选择理由**：
- 数据量小（几十到一百多个节点），不需要数据库
- 数据相对稳定，不需要频繁修改
- 无复杂查询需求，只需要读取完整数据
- 个人使用，无并发需求
- 零配置、零依赖，易于部署和备份
- 版本控制友好，可以用Git管理
- **关键优势**：避免浏览器安全限制，支持直接打开HTML文件（file://协议）时正常加载数据

**文件结构示例**：
```
project/
├── index.html          # 主页面
├── css/
│   └── style.css       # 样式文件
└── js/
    └── app.js          # 业务逻辑（包含内嵌的graphData数据）
```

### 5.3 性能要求
- 图谱渲染流畅，支持100+节点
- 悬停响应时间 < 100ms
- 页面加载时间 < 3s
